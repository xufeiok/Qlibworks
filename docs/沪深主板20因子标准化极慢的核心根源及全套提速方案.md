# 沪深主板20因子标准化极慢的核心根源及全套提速方案

# 一、沪深主板 20 因子标准化极慢的核心 4 个根源（按概率排序）

## 1\. 循环逐日期 / 逐股票串行计算（最常见）

量化新手写法：for 循环遍历每一天，再循环每支股票，20 个因子分别做 Zscore/Rank/MAD。
沪深主板常年约 1300–1600 只个股，回测区间 10 年 = 2500 交易日，单循环就是 `2500×1500×20` 次运算，纯 Python 循环性能暴跌。
树模型预处理普遍用 DataFrame，逐行循环是性能杀手。

## 2\. 未做截面向量化，全市场全局计算

错误操作：不按`date`分组，一次性全量数据计算均值 / 中位数 / 分位数；
全局统计会读取全量内存，分组筛选、广播对齐开销巨大；
Rank 标准化尤其吃性能，全局 rank 远慢于每日分组 rank。

## 3\. 数据存储与索引低效

- 无时间 \+ 股票复合索引，分组`groupby("date")`每次全表扫描；

- 用普通 csv 循环读取，未转 parquet/feather 二进制；

- 保留停牌、ST、退市冗余样本，数据量膨胀 2–3 倍；

- 20 个因子分开存多张表，反复 merge 拼接，IO 耗时远超计算。

## 4\. 标准化算法本身低效

1. 普通 Z\-score：`groupby+apply`自定义函数，apply 是串行黑盒；

2. MAD 稳健标准化：循环计算 MAD、上下限截断，双重循环；

3. Rank 使用`rank(method="average")`慢速模式；

4. 中性化线性回归用 sklearn 逐天循环拟合 OLS，未向量化矩阵运算。

# 二、全套提速方案（分 IO、向量化、算法、并行四层，落地立刻见效）

## （一）底层数据 IO 提速（见效最快，3–10 倍提升）

1. **存储格式替换**
丢弃 CSV，全部转 `feather / parquet`，列式存储，读取因子只加载需要的 20 列，不用全量读几百个因子。

2. 建立固定复合索引
`df.set_index(["date","stock_id"], inplace=True)`，分组、筛选、对齐 O \(1\) 检索，groupby 速度提升 5 倍以上。

3. 提前过滤冗余样本，减少计算量
预处理第一步直接剔除：ST/\*ST、停牌、上市 \< 6 个月、日均成交额后 20% 个股，直接砍掉 30%–50% 数据量，标准化计算量同步减半。

4. 单表整合因子
一张表：date, stock\_id, factor1,factor2…factor20，避免每日 merge 多因子表。

## （二）向量化重写标准化逻辑（核心提速，10–100 倍，彻底抛弃 for/apply）

### 规则：禁止 `df.groupby("date").apply(自定义标准化函数)`

全部用 `groupby.transform` 向量化聚合，transform 输出和原表同长度，无循环。

### 1\. 截面 Rank 标准化（推荐，速度最快，替代 Z\-score）

```python
# 每日截面秩变换，映射[-1,1]，纯向量化，无循环
ranked = df[factors].groupby("date").rank(pct=True)
df[factors] = ranked * 2 - 1
```

rank 自带向量化 C 实现，比手写 Z\-score 快几十倍。

### 2\. 普通截面 Z\-score（向量化写法）

```python
group = df.groupby("date")[factors]
mean = group.transform("mean")
std = group.transform("std")
df[factors] = (df[factors] - mean) / std
```

不用 apply，全底层 C 运算。

### 3\. MAD 稳健标准化（向量化封装，替代逐天循环）

先分组 transform 取中位数、MAD，广播计算上下限，一次性截断 \+ 标准化，避免循环拟合。

### 4\. 缩尾前置向量化

上下 5 倍 MAD 缩尾同样用 transform 批量算出阈值，`where`批量截断，不循环单只股票。

## （三）分组并行加速（大数据长周期专用）

两种并行方案二选一，不要叠加：

1. **modin /swifter 自动并行 DataFrame**
替换 pandas，groupby 自动多线程，仅需改一行导入，原有代码几乎不用动；适合单 CPU 多核机器。

2. **按年份分块多进程**
将 2500 交易日切分成 2–3 年一块，multiprocess 并行处理不同时间段，每块独立做标准化，最后拼接；
注意：不要单天多进程，进程创建开销会抵消收益。

## （四）中性化额外提速点（20 因子必看）

你 20 个因子都要行业 \+ 市值中性，逐天循环 sklearn OLS 极慢：

1. 用`statsmodels`矩阵批量回归或`numpy.linalg`向量化残差计算；

2. 或使用预封装向量化中性库（qlib 内置 neutralize 底层矩阵运算）；

3. 优先 Rank 标准化后再中性，残差波动更小，迭代更快。

## （五）硬件 \& 参数小优化

1. pandas 开启 numba 加速：对自定义 MAD 截断逻辑加`@numba.jit`，数值循环提速 50 倍；

2. 关闭 rank 多余参数：`rank(pct=True, method="min")`比默认 average 更快；

3. 内存优化：因子统一 float32 存储，float64 内存占用翻倍，分组计算更慢。

# 四、最简落地优化步骤（直接按顺序改）

1. 因子数据合并单张 feather 表，设 date \+ 股票索引；

2. 读取后第一行过滤 ST、次新、低流动性股票，缩小数据集；

3. 放弃 apply 循环，全部 groupby\.transform 向量化计算；

4. 优先使用截面 Rank 标准化替代 Z\-score，计算开销最低；

5. MAD 缩尾、中性化全部用矩阵向量化，不逐天循环拟合；

6. 超长回测区间使用分块多进程并行处理。
