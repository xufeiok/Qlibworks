# Qlib特征数据分析报告

## 问题总结

在对 `e:\Quant\Qlibworks\qlib_data\features` 目录下的特征数据进行检查后,发现以下问题:

### 1. **数据完整性问题**

#### ✅ **正常特征** (数据完整,0值比例正常)
以下特征数据质量良好:
- `close.day` - 收盘价
- `open.day` - 开盘价  
- `high.day` - 最高价
- `low.day` - 最低价
- `amount.day` - 成交额
- `volume.day` - 成交量
- `vwap.day` - 成交量加权平均价
- `factor.day` - 复权因子

这些特征:
- **0值比例**: 0% (正常)
- **NaN比例**: 因股票而异,通常<10% (停牌或缺失数据)
- **数据长度**: 2000-6500不等,取决于股票上市时间

#### ⚠️ **高0值特征** (部分数据缺失)
以下特征存在大量0值(50%-90%+):

**财务指标类**:
- `basic_eps_yoy.day` - 基本每股收益同比增长率
- `dt_netprofit_yoy.day` - 扣非净利润同比增长率  
- `eps.day` - 每股收益
- `grossprofit_margin.day` - 销售毛利率
- `netprofit_margin.day` - 销售净利率
- `netprofit_yoy.day` - 净利润同比增长率
- `tr_yoy.day` - 营业收入同比增长率
- `ocfps.day` - 每股经营性现金流
- `roa.day` - 总资产收益率
- `roe_ttm.day` - 净资产收益率(TTM)
- `current_ratio.day` - 流动比率
- `debt_to_assets.day` - 资产负债率
- `inv_turn.day` - 存货周转率

**原因**: 财务指标通常**按季度发布**,一年只有4个时间点有数据,其余时间填充0。这是**正常现象**。

**市场数据类**:
- `circ_mv.day` - 流通市值
- `total_mv.day` - 总市值
- `pb.day` - 市净率
- `pe.day` - 市盈率
- `pe_ttm.day` - 市盈率(TTM)
- `ps.day` - 市销率
- `ps_ttm.day` - 市销率(TTM)

**原因**: 这些指标在某些情况下(如净利润为负、停牌等)会显示为0。

#### ❌ **严重缺失特征** (几乎全部为0)
以下特征98%+的数据都是0:

- `dv_ttm.day` - 股息率(TTM)
- `eps_forecast.day` - 每股收益预测
- `north_hold.day` - 北向资金持股
- `pledge_ratio.day` - 质押比例
- `rzye.day` - 融资融券余额
- `stk_holdernumber.day` - 股东户数

**原因**: 这些数据可能是:
1. **数据源未提供** - 需要特定的数据权限
2. **时间序列较短** - 如北向资金数据近年才有
3. **需要额外配置** - 某些指标需要特殊计算

### 2. **数据长度不一致**

不同股票的数据长度差异很大:
- **000001.sz**: 3964个交易日
- **000003.sz**: 2669个交易日  
- **000004.sz**: 3933个交易日
- **000005.sz**: 6449个交易日

**原因**: 股票**上市时间不同**,这是**正常现象**。

### 3. **NaN值分布**

NaN值通常出现在:
- 股票停牌期间
- 财务数据未披露期间
- 新上市股票缺少历史数据

典型的NaN比例:
- **正常股票**: 0-5%
- **长期停牌股票**: 可达30-40%
- **新股票**: 可能更高

## 建议

### 1. **对于正常使用**
- ✅ 可以正常使用: `close`, `open`, `high`, `low`, `volume`, `amount`, `vwap`, `factor`
- ⚠️ 使用财务指标时注意: 数据是季度性的,需要做前向填充或处理0值
- ❌ 高0值特征需要确认数据源是否支持

### 2. **改进数据质量**
如需完整的市场指标数据,建议:
1. 检查数据下载配置
2. 确认数据源是否提供这些指标
3. 查看是否需要更新数据
4. 对于财务指标,考虑使用前向填充(ffill)策略

### 3. **数据预处理建议**
在使用这些特征前:
```python
# 处理0值
data = data.replace(0, np.nan)  # 将0转为NaN
data = data.ffill()  # 前向填充

# 或者直接过滤
valid_data = data[data != 0]
```

## 结论

1. **基础量价数据**(close/open/high/low/volume/amount)**质量良好**,可以放心使用
2. **财务指标数据**存在大量0值是**正常现象**(季度发布),需要适当处理
3. **部分高级指标**(dv_ttm/north_hold/rzye等)**数据严重缺失**,需要确认数据源

**总的来说**: 数据是完整的,但需要根据不同特征的特点进行适当的预处理。
